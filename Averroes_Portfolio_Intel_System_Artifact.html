3. The Era Model (Critical Concept)

Portco Alpha's Management Accounts evolved dramatically over 28 months (Nov 2023 – Feb 2026). The parser branches by "era" because the available Excel sheets and their layouts differ significantly between time periods.

EraMonthsFilesKey Sheets AvailableParser
Era 1 Nov 2023 – Oct 2025 24 Summary (with trailing space drift), Headcount, Ecommerce P&L, EMS P&L. Covers both FY24 (Nov 23–Oct 24) and FY25 (Nov 24–Oct 25) with layout drift handled by label scanning. era1_parser.py
Era 2 Nov 2025 – Dec 2025 2 + P&L Detail, Balance Sheet, GL Covenants, Averroes Guard Rails, Cosmo Portal Upload era2_parser.py
Era 3 Jan 2026 – Feb 2026 2 + Financial KPIs, Revenue Waterfall, Customer Numbers, Cash Flow, KPI data, P&L Summary era3_parser.py
FY24 Layout Drift (handled by Era 1 parser):
FY24 compact (Nov 23 – ~May 24): No EBITDA Less Capex row; Tech MRR/Cash/NWC start at R20. Col 10 = Variance (£), NOT Prior Year.
FY24 extended (Jun 24 – Oct 24): Capitalised Dev + EBITDA Less Capex rows inserted; Tech MRR at ~R28. Col 10 = Variance (£), NOT Prior Year.
FY25 (Nov 24 – Oct 25): Same extended layout, adds LTM Tech MRR row. Col 10 = Prior Year.
The parser detects these variants by scanning R3 C10 header for "Prior"/"PY" vs "Variance", and label-scanning the bottom section (R19+) rather than using hardcoded row numbers.

4.1 Common Utilities — parsers/common.py

Every parser returns rows shaped as: {period, kpi, value, value_type, business_line, sheet, source_cell}

def period_from_filename(file_name):
    """Best-effort period extraction from filename.
    Handles: 'FY25 Management Accounts - June 25.xlsx' -> 2025-06-01
    '4. FY26 Management Accounts - February 26.xlsx' -> 2026-02-01"""
    # Regex matches <month_name> <2-digit-year>
    m = re.search(r'(january|...|dec)\s*[- ]?\s*(\d{2})\b', name)
    if m:
        mo = months[m.group(1)]
        yr = 2000 + int(m.group(2))
        return f"{yr:04d}-{mo:02d}-01"

def safe_number(v):
    """Coerce cell value to float, or None if not numeric."""
    if isinstance(v, (int, float)) and not isinstance(v, bool):
        return float(v)
    return None

def find_sheet(wb, target_name):
    """Find sheet by name, tolerating trailing space and case drift."""
    target_n = target_name.strip().lower()
    for s in wb.sheetnames:
        if s.strip().lower() == target_n: return s
    return None

def row(period, kpi, value, value_type='actual', business_line=None, sheet=None, source_cell=None):
    """Build a canonical row dict."""
    return {
        'period': period, 'kpi': kpi, 'value': float(value),
        'value_type': value_type, 'business_line': business_line,
        'sheet': sheet, 'source_cell': source_cell,
    }

4.2 Era 1 Parser — parsers/era1_parser.py (FY24 + FY25)

Key design: Era 1 covers 24 files across two fiscal years (FY24: Nov 2023–Oct 2024, FY25: Nov 2024–Oct 2025). The P&L section (R5–R18) is stable and hardcoded. The bottom section (R19+) uses label scanning because row positions drift between FY24 compact, FY24 extended, and FY25 layouts. Prior Year column is detected dynamically from R3 header.

4.2.1 Prior Year Column Detection

def _has_prior_year_column(ws):
    """Check R3 C10 header: FY25 has 'PY'/'Prior Year', FY24 has 'Variance'."""
    header = str(ws.cell(3, 10).value or '').strip().lower()
    return 'prior' in header or header == 'py'

5. Silver & Gold Transform Logic

5.1 silver_gold_v2.py — Unit Normalization

CRITICAL UNIT CONVENTION:
Era 3 Tech MRR & Services MRR are in raw GBP (e.g., 467,135). All other P&L/BS values are in GBPk (thousands). The silver layer normalizes Era 3 MRR from raw GBP to GBPk by dividing by 1000. 
RAW_POUND_KPIS = {"TECH_MRR", "LTM_TECH_MRR", "SERVICES_MRR", "LTM_SERVICES_MRR"}

def normalize_unit(kpi, value, era):
    """Normalize MRR to GBPk (Era 3 source is raw GBP)."""
    if kpi in RAW_POUND_KPIS and era == "era3":
        return value / 1000.0
    return value

5.3 Gold Pivot — pivot_to_gold()

Pivots long silver into wide gold (one row per portco × period). Key behaviors:

  • FY Calculation: FY runs Nov–Oct. Month ≥ 11 → next FY. fy_month_num = month−10 for Nov+, month+2 for Jan–Oct.
  • CASH_FIN_KPI conversion: cash_balance_budget and cash_balance_prior_month are converted from raw GBP to GBPk via _div_k().
  • Indicative EV: Also converted to GBPk via _div_k().
  • YTD Cumsum: EBITDA YTD is filled via cumulative sum within each FY, but only where P&L Summary didn't already provide a value (uses fillna not overwrite).
  • Revenue growth %: Derived as (actual / prior_year - 1) * 100.

6. Cloud Function Pipeline

6.1 functions/ingest/main.py — Complete Flow

@functions_framework.cloud_event
def process_file(cloud_event):
    # 1. Extract portco_id from GCS path (first path segment)
    # 2. Filter: accept any .xlsx/.xls/.xlsm, skip ~$ and . prefixed files
    if not basename.lower().endswith((".xlsx", ".xls", ".xlsm")):
        return  # skip non-Excel
    if basename.startswith(("~$", ".")):
        return  # skip temp/hidden files

    # 3. Download blob bytes from GCS (with size logging)
    file_bytes = blob.download_as_bytes()
    print(f"[ingest] downloaded {len(file_bytes):,} bytes")

    # 4. PARSE (with try/except + traceback logging)
    try:
        parsed_rows = parse_alpha_ma(file_bytes, basename)
    except Exception as e:
        print(f"[ingest] ERROR: {e}"); traceback.print_exc()
        return
    # Logs: era, periods extracted, row count

    # 5. BRONZE - Idempotent: DELETE by file_name, then INSERT
    write_bronze(parsed_rows, portco_id, basename)

    # 6. SILVER - build_silver_from_parsed() + write
    silver_df = build_silver_from_parsed(parsed_rows, source_file=basename)
    write_silver(silver_df, portco_id, basename)  # DELETE by source_file, then INSERT

    # 7. GOLD - pivot_to_gold() + write
    gold_df = pivot_to_gold(silver_df, portco_id=portco_id)
    write_gold(gold_df, portco_id)  # DELETE by portco_id+period, then INSERT

7.3 Formatter Functions

def fmt_gbp_k(val, decimals=2):
    """Input is in GBPk. 880 (GBPk) -> 'GBP0.88 M', 50 (GBPk) -> 'GBP50 k'"""
    raw_val = val * 1000              # convert to raw GBP
    abs_val = abs(raw_val)
    sign = "-" if raw_val < 0 else ""
    if abs_val >= 100_000:            # >= GBP100k -> show as M
        return f"{sign}GBP{abs_val / 1_000_000:,.{decimals}f} M"
    elif abs_val >= 1_000:            # >= GBP1k -> show as k
        return f"{sign}GBP{abs_val / 1_000:,.0f} k"

def fmt_gbp(val, decimals=2):
    """Input is in raw GBP. 467135 -> 'GBP467 k', 5605620 -> 'GBP5.61 M'"""
    abs_val = abs(val)
    sign = "-" if val < 0 else ""
    if abs_val >= 1_000_000:          # >= GBP1M -> show as M
        return f"{sign}GBP{abs_val / 1_000_000:,.{decimals}f} M"
    elif abs_val >= 10_000:           # >= GBP10k -> show as k
        return f"{sign}GBP{abs_val / 1_000:,.0f} k"

13. Known Issues & Gotchas

Parser Resilience (v3.0): Era 1 parser now uses label scanning for the bottom section (Tech MRR, Cash, NWC, etc.) and BL P&L breakdowns, making it resilient to row-position drift across FY24/FY25 variants. Era 2/3 parsers still use some hardcoded positions (e.g., R18 for NWC in Balance Sheet, R6C6 for TCV). Always verify dashboard values against source Excel when a new MA template is received.
Cloud Function File Filter (v3.0 — FIXED): Previously only accepted filenames containing "Management Accounts" or "MAfile". Now accepts any .xlsx/.xls/.xlsm file, skipping only temp files (~$) and hidden files (. prefix). This fixed the issue where FY24 files uploaded to GCS didn't trigger processing.
P&L Summary Trailing Space: The sheet name is 'P&L Summary ' (with trailing space) in some files. The find_sheet() helper handles this by stripping whitespace, but direct wb['P&L Summary'] lookups will fail.
Balance Sheet NWC Label: Net Working Capital at R18 has its label in column A, not column B like all other BS labels. The parser checks both columns.
Financial KPIs Block Exceptions: Three blocks deviate from the standard PY/Budget/Actual pattern: Indicative EV (single value), S&M Efficiency (YTD-only), and Time to Value (extra "excl. blocked" row at +4). These have explicit if kpi == ... handlers.
Services MRR/ARR Gap: SERVICES_MRR is only available from the Financial KPIs sheet (Era 3, Jan 2026+). It does not exist in Era 1 or Era 2 data. Therefore SERVICES_ARR (= Services MRR × 12) is only populated for Era 3 months. Do NOT derive/estimate Services MRR from revenue — only use when explicitly present in source data.
Fallback CSV Drift: The local CSV fallback at dashboard/gold_kpi_monthly.csv is only refreshed during deploy_v2.py runs. If new data is ingested via the Cloud Function, the CSV falls behind. The BQ path should always be preferred.

15. Quick-Start for New Developer / AI

To resume development on this project, follow these steps:
  1. Clone: git clone https://github.com/averroescapital/averroes-intel.git
  2. Install deps: pip install -r requirements.txt and pip install openpyxl
  3. Set GCP credentials: export GOOGLE_APPLICATION_CREDENTIALS=/path/to/key.json
  4. Run dashboard locally: streamlit run pe_app.py
  5. To backfill/deploy: python3 deploy/deploy_v2.py --key /path/to/key.json
  6. To add a new MA file: Upload to GCS bucket (portco-alpha/ma-files/); Cloud Function auto-triggers. Or place in raw_ma_files/ and run deploy.
  7. To reprocess all files: Hit ⚙️ Reprocess GCS Files button in dashboard sidebar (copies blobs in-place to re-trigger Cloud Function)
  8. To modify parser: Edit functions/ingest/parsers/era3_parser.py (canonical), then backfill
  9. To add new gold columns: Add to bq_schemas/silver_gold_v2.sql + silver_gold_v2.py:pivot_to_gold() + pe_app.py:harmonize_v2_columns()
